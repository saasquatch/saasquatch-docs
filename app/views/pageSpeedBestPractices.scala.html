Page Load Times Best Practice Guide

squatch.js is loaded asynchronously to keep your page load fast

Keep it in the body. Place the <script> tag examples that we provide right before the </body> tag, or at the very least, keep it out of the <head> tag. This will make sure that the squatch.js is the last script to execute, and won't delay other scripts on your page.

Use the default onload function. By default our javascript library will only load after the rest of your page is fully loaded. Don't directly reference the squatch.min.js library, because you might end up making it load synchronously, which means that it will delay the rest of your page from loading. Instead, use the default load function in our script examples.

Use timeout. If you have a lot of assets to load in your app, you can also choose to delay the initialization of squatch.js by some fixed time. New to javascript timeouts? Check out this StackOverflow post on the subject. for advanced javascript programmers

We serve all of our assets, like images, javascript and CSS, from the Amazon Cloudfront CDN. This makes sure that your customers have the lowest latency possible. We know that a quick and responsive app can greatly improve your engagement rate, so we're constantly developing new ways to reduce file size and improve user-perceived latency.
